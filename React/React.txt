1. First Reaction
# The Rebirth
- React is a JavaScript library used for building websites with reusable components. It lets you create a multi-page website and not have to update things repeatedly in different files.

Some of React's most useful features include:

+ An interesting flavor of JavaScript called JSX.
+ An efficient way to update parts of your site.
+ It's fast and scalable!

Before React, updating your website meant changing the same element in every file. With React, you change a single element in one place and that change applies everywhere!

03. Writing React
# React Syntax
When we import React from the "react" library, we can use JavaScript Syntax Extension (or "JSX").
let heading = <h1 id="welcome-heading">Welcome!</h1>
(Notice anything familiar? Wait, that looks just like an HTML element... inside JavaScript?)

- JSX is really more like "syntactic sugar" because it looks like HTML, but it's actually JavaScript! 💡
- We can define JSX either in a .js file or a .jsx file. For this course, we're using .js files!

When you run your code, JSX gets transpiled, or translated into plain JavaScript and then read by the browser.

// Before being transpiled
let heading = <h1 id="welcome-heading">Welcome!</h1>;

// After being transpiled
let heading = React.createElement("h1", { id: "welcome-heading" }, "Welcome!");

Fortunately, JSX simplifies things with its friendly syntax!


04. Building Blocks
# What Apps Are Made Of
- Another core feature of React are components.
- Components are defined like functions that return JSX content.
- If JSX is like the "language" of React, then components are like the "building blocks" of the website's user interface.

import React from "react";

export default function Greetings() {
  return (
    <div>
      <h1>Hello, World!</h1>
      <p>Nice to meet you. 😊</p>
    </div>
  )
}

In the example above:

+ We import React to be able to use JSX and write components.
+ Then, we define the component as a JavaScript function that returns something.
+ Lastly, we return a heading and paragraph wrapped in a <div> element.

Note: Your returned JSX must be wrapped in a single element.


While defining the <Greetings> component, we attached the export default keywords. This lets us use the component in other files!

To add an imported component to another component's return statement:

import Greetings from "path/to/Greetings.js";

export default function App() {
  return <Greetings />
}


When referencing another component, we write like a self-closing HTML element with a / slash.

Also, the export default keywords were used for the <App> component. This means it can be used in another file!

# Conditional JSX
JSX can be assigned to variables as a value or it can be returned from a function. . But we can also use conditional statements to determine exactly what JSX should be displayed.

Imagine signing into a website. We could use an if/else statement to conditionally assign JSX to a variable that points to the right page markup:
let page = null;
let signedIn = true;

if (signedIn) {
  page = <div>User dashboard</div>
} else {
  page = <div>Sign-in page</div>
}

export default function App() {
  return (
    <div>
      {page}
    </div>
  )
}

Note: By the way, JSX comments look unique and interesting: {/* Right?! */}

+ If you want what is inside the JSX tags to be conditionally set, we can use a ? : ternary operator:

let page = <div>{signedIn ? {/* User dashboard */} : {/* Sign-in page */}}</div>

+ If no else condition is present, we can use the && "and" or || "or" operators:

let page = <div>signedIn && {/* User dashboard */}</div>

# Attributes & Names
const selfie = <img src="username/camera/recents/today.png" alt="a selfie" />

- This selfie variable contains a JSX image with two attributes: src and alt. Any valid HTML and their associated attributes are supported in JSX…mostly.

- One of the most widely used attributes in HTML is class:
<button class="login-button">
  Submit
</button>

- However, the JSX-equivalent to this is className:
<button className="login-button">
  Submit
</button>

In JavaScript, "class" is actually a reserved keyword. Therefore, we use className!

09. Hot Takes Pt. 1
# Components
Components are pieces of JSX that make up most of the website. They can be anything from labels to buttons to navigation menus.

Below is an example of a component defined with a function keyword.
import React from "react";

export default function FunctionComponent() {
  return (
    <div>
      {/*This is a comment*/}
      <h1>Here's some markup!</h1>
    </div>
  )
}

+ The name of the component must begin with a capital letter, and be in "PascalCase".
+ The JSX returned by a component is bundled under a single element and rendered as a single part, or component, of a website.

# Using Components
First, we can set them up to be used in other components with the following:

export default function FunctionComponent() {


}
----> We use the export default keywords so the component can be used in other files.


Then, to import the component for use in another component file:

import FunctionComponent from "file/path/to/FunctionComponent.js";


We can refer to them the same way we refer to any JSX, with angle brackets!
function App() {
  return (
    <FunctionComponent />
  )
}

Every React application must be rendered from a root component. It is usually done like this:
import React from "react";
import { createRoot } from "react-dom/client";

export default function App() {
  return (
    <div>
      {/* Returned JSX here */}
    </div>
  )
}

const container = document.getElementById("root");

const root = createRoot(container);

root.render(<App />);


First, we import the createRoot() at the top of the .js file. Then, we define our component (usually named <App> or <Home>).
