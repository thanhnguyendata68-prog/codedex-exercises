VII. Data Fetching
# Data Fetching
- We can fetch data from outside our applications! It's what makes for dynamic and engaging user experiences. When you're on the feed of your favorite social media app, the app is most likely fetching data from somewhere else!

- In this chapter, we'll learn how to fetch and use data from outside sources.

# fetch()
- Before we fetch data, we need a component to render first. This makes the useEffect() hook the ideal place since our side effect code runs after its host component renders.

- Think of fetching data in a component like setting up a table. Before you add plates and food dishes, you first need a table to set them on.

In the example below, we get cat photos with the fetch() function. 🐱
import React, { useState, useEffect } from "react";

export default function CatImages() {
  const [catImages, setCatImages] = useState([]);

  useEffect(function() {
    async function fetchCatImages() {
      const response = await fetch("https://api.thecatapi.com/v1/images/search?limit=10");
      const data = await response.json();
      setCatImages(data);
    };

    fetchCatImages();
  }, []);

  return (
    <div>
       {catImages.map(function(catImage, index) {
        return <img width="100" key={index} src={catImage.url} />
       })}
    </div>
  );
}

Let's break this down:

+ The useEffect() hook is used with an effect function and empty dependency array.
+ Inside, we define an asynchronous fetchCatImages() function.
+ This function uses a fetch() function to get cat photo data via URL.
+ The async/await keywords ensure we get data before fetchCatImages() is done.
+ We update our catImages state with the fetched data.

33. Now Loading
# Promises
- Let's take a closer look at fetch(). This function returns a special object called a Promise.

- A Promise is an object that we get back from an async operation, like fetching data. They have three statuses:
+ Once run, the Promise is "pending" while fetching data.
+ If it successfully fetches data, the Promise is "fulfilled".
+ If it can't fetch data for some reason, the Promise is "rejected".

import { useState, useEffect } from "react";
export default function DogImage() {
  const [dogImage, setDogImage] = useState("");
  useEffect(function() {
    async function fetchDogImage() {
      // Promise begins as "pending"
      const response = await fetch("https://dog.ceo/api/breeds/image/random");
      const data = await response.json();
      console.log(data.message);
      setDogData(data.message);
    }
    fetchDogImage();
  }, [])
  return <img width="300" src={dogImage} />
}

--> When we run the fetch() function, the Promise object is "pending". Because of async/await, the function doesn't continue until the Promise is fulfilled or rejected. Same goes for when we get response object data with .json().

# Loading State
- If we're fetching data and it takes some time for a Promise to be fulfilled (or rejected), it's good practice to update our UI that lets the user know we're still fetching.

A way to do this would be with some kind of loading state variable. Check out the following example:
import { useState, useEffect } from "react";

export default function App() {
  const [loading, setLoading] = useState("");
  
  useEffect(function() {
    // Set the data to "Data pending..." initially
    setLoading("Data pending... 🔄");

    async function fetchData() {
      const response = await fetch("https://jsonplaceholder.typicode.com/posts/1");

      const data = await response.json();
      setLoading("Data has been fetched! ✅");
      return data;
    }

    // Timer to update the state after 3 seconds
    const timer = setTimeout(function() {
      // Fetch data after 3 seconds
      fetchData();
    }, 3000);

    // Clear the timer to avoid memory leaks
    return function() {clearTimeout(timer)};
  }, []);

  return (
    <div>
      {loading}
    </div>
  );
}

After the component's first render, our useEffect() code updates the loading state and renders "Data pending... 🔄" to the screen. Then, we perform an async data fetch and update the loading state again after finishing the fetch. To help visualize this,, we use setTimeout() to delay the fetch by 3 seconds:

34. I Am Error
# Rejected Fetches
- When fetching data from an outside source, there's a chance we may not be successful. As mentioned, the returned Promise object can have a "rejected" status. There are a few reasons this may happen:

+ It took too long to fetch the data and we timed out.
+ Something is wrong with the data source or it no longer exists.
+ We may have misspelled something in the URL we passed to fetch().

---> As a result, we get an error but may never know about it if we don't handle it. In a React app, we can handle errors from fetching data and update the rendered JSX as a result.

# Catching Errors
One common way to handle data fetch errors in React is with a try/catch statement:

try {
  const response = await fetch('https://api.example.com/data');
  if (response.ok === false) {
    throw new Error('Failed to fetch data');
  }
  const data = await response.json();
} catch (error) {
  console.log(error.message);
}

In the try-block:

+ We fetch our data as usual, and store it in a response variable.
+ If the response.ok property is false, we throw a new error with a custom message.
+ Otherwise, store the response contents in a data variable.

---> If we throw an error, we go straight to the catch-block. There, we pass an error object and log its .message property to the console. This property refers to the custom error message we set in the try-block.

## Rendering Errors
- Suppose we encountered an error while fetching data in our React code. We can use the useState() hook and conditional rendering to display the error message to the user:

import { useState, useEffect } from "react";

export default function ErrorExample() {
  const [error, setError] = useState(null);
  
  useEffect(function() {
    async function fetchData() {
      try {
        const response = await fetch("https://jsonplaceholder.typicode.com/posts/1ERR");
        if(response.ok === false) {
          throw new Error("Failed to fetch data");
        }
        setError(null);
      } catch(err) {
        setError(err.message); // Set error message here
      }
    }
    fetchData();
  }, []); // Add an empty dependency array here to ensure the effect runs only once

  return (
    <div>
      {error && <div>{error}</div> || <h1>No errors!</h1>}
    </div>
  );
}

--> We initialized an error state to null since an error hasn't occurred yet. Inside useEffect(), we defined an async fetchData() function that uses a try/catch statement to throw and handle errors that may occur. If our returned Promise is "fulfilled", we keep the error set to null. Otherwise, we update the error with the message we set.

- In the return statement, we use the && and || operators to conditionally render either the error message or a heading. The output looks like this:

useEffect(function() {

  }
  fetchData();

With this data source, we can access the actual data via a .results property that points to an array of 10 Star Wars character objects.
